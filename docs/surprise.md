# Cerulean — Surprises & Learnings

Purpose: Record unexpected discoveries, gotchas, and insights that emerged during development. This file feeds the user's continuous improvement loop across projects.

## How To Use
- Only log genuinely surprising or non-obvious discoveries — things a competent engineer wouldn't already know or find via a quick search.
- Do NOT log routine facts, standard library behavior, or common gotchas you'd learn during initial setup.
- This file does not need an entry after every session. Update it only when something genuinely surprising is learned.
- Keep entries concise: what surprised you, why it matters, and what to do about it.
- Only edit a past entry if a new discovery invalidates or changes it.
- Do not duplicate information from the changelog — this is for *insights*, not a record of changes.
